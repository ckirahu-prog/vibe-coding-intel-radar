# 方法论速查表（内嵌 Prompt · 勿 @ skill 文件）

> 从 `.cursor/skills/` 提炼的硬规则。Automation 只读本节，不读取 skill 全文。

---

## CHEAT-ICP · 模块 A1 人群与场景

- **必填字段**：人群（谁）/ 场景（何时何地）/ JTBD（要完成什么）/ 痛点（摩擦句 ≤40 字，verbatim 优先）
- **无 URL + 无原文引用** → 不得进 A1 或 A2
- **形态标签**（工具/服务/内容）无原文支撑 → 标【推断】
- **地域**：中国本土优先；纯海外 → A2 或 A4，不进 A1

---

## CHEAT-REVIEW · 模块 A1/A2 社区 friction

- 从社区帖抽 pain：**直接引用 ≤40 字**，不扩写、不美化
- 优先 **unprompted 抱怨**（用户主动吐槽，非记者转述）
- 1–2 星式 friction = 强；宏观政策/科技新闻无具体摩擦 → **跳过**
- **媒体 alone**（无社区 friction 句）→ 不得 A1
- A2 保留：有 friction 但缺场景/引用、纯海外待迁移、只有 workaround

## CHEAT-COMP · 模块 A4 竞品

- **简表 4 列**：产品 | 优点 | 可优化点 | 来源 URL
- 每个 A1 簇 **0–2 个**参照；无 URL 写「未知」，禁止凭记忆编
- **禁止**扩写为 5 家完整竞品长报告
- 竞品空白 ≠ 无需求（须写清，不得断言蓝海）
- 顺序：仓库内已有 → 再联网（计入每周 ≤3 次总配额）

---

## CHEAT-MOM · 附录 D4 验证行动

- 问 **过去行为**，不问「会不会买/会不会用」
- 好问题：「上次遇到这个问题你做了什么？」「目前怎么解决？」
- 坏问题（禁用）：「你觉得这个 idea 好吗？」「你会付多少钱？」
- **单源 A1 簇**：D4 副业行动第一步 = **找第二源**（具体平台+关键词）
- 所有 D4 行动须在 **10–20 小时/周**内可完成；需全职 → 标【高投入】或只进 A2

---

## CHEAT-CROSS · 交叉验证与 enrichment 素材

- **单源 A1** 可保留，但必须标「**单源，待交叉验证**」
- **升级为双源 A1** 须满足：第二独立 URL 出现**同类 friction**（非记者转述、非同一帖转发）
- 第二源可来自：`data/raw/` 不同平台、`data/enriched/` 中 last30days 摘录、`manual-urls.yaml` 手补链接
- **enrichment 不能替代原文**：A1 仍须保留首条 verbatim friction ≤40 字 + URL；第二源单独列出
- enrichment 只作**佐证**，不得用 brief 摘要替代社区原文进 A1
- 若 enrichment 与首源 friction **不一致** → 保持单源或降级 A2，不得强行合并
- 周六 Automation **禁止**现场跑 `/last30days` 或 Agent-Reach；只读仓库内已有 `data/enriched/`

## CHEAT-GENRE · 模块 B2 热门题材

- **热门依据必填**：信号类型 + 平台（Steam/itch 为主）+ 日期 + URL；无则不进 B2 主表
- **时间窗**：近 3–6 个月；无近期信号 → 观察池 0–1 条或跳过
- **strict solo scope（必须全部满足才进 B2）**：
  - 2D 或轻量 3D；核心循环目标 8–12 周可玩 demo
  - **排除**：3A 开放世界、强联机/live ops、重度多人、大量手工关卡
- **每簇必填**：核心循环 | 吸引原因 | 单人 scope（做/不做）| AI 可介入环节
- 手游/主机：仅 **跨平台参照** 一句，不作主信号
- 联网补商店页：计入与 A4 **合计 ≤3 次/周**
