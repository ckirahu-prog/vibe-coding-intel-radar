## 方法论速查表（内嵌 · 执行前通读）

> 从 `.cursor/skills/` 提炼。Automation **不读取** skill 全文。

### CHEAT-ICP · 模块 A1 人群与场景

- **必填字段**：人群（谁）/ 场景（何时何地）/ JTBD（要完成什么）/ 痛点（摩擦句 ≤40 字，verbatim 优先）
- **无 URL + 无原文引用** → 不得进 A1 或 A2
- **形态标签**（工具/服务/内容）无原文支撑 → 标【推断】
- **地域**：中国本土优先（70/30）；纯海外 → A2 或 A4，不进 A1

### CHEAT-REVIEW · 模块 A1/A2 社区 friction

- 从社区帖抽 pain：**直接引用 ≤40 字**，不扩写、不美化
- 优先 **unprompted 抱怨**（用户主动吐槽，非记者转述）
- 宏观政策/科技新闻无具体摩擦 → **跳过**
- **媒体 alone**（无社区 friction 句）→ 不得 A1
- A2 保留：有 friction 但缺场景/引用、纯海外待迁移、只有 workaround
- 中文源目标：**社区 60% + 媒体 40%**（媒体 alone 不进 A1）

### CHEAT-COMP · 模块 A4 竞品

- **简表 4 列**：产品 | 优点 | 可优化点 | 来源 URL
- 每个 A1 簇 **0–2 个**参照；无 URL 写「未知」，禁止凭记忆编
- **禁止**扩写为 5 家完整竞品长报告
- 顺序：仓库内已有 → 再联网（计入每周 ≤3 次总配额）

### CHEAT-MOM · 附录 D4 验证行动

- 问 **过去行为**，不问「会不会买/会不会用」
- **单源 A1 簇**：D4 副业行动第一步 = **找第二源**（具体平台+关键词）
- 所有 D4 行动须在 **10–20 小时/周**内可完成

### CHEAT-GENRE · 模块 B2 热门题材

- **热门依据必填**：信号类型 + 平台（Steam/itch 为主）+ 日期 + URL
- **时间窗**：近 3–6 个月
- **strict solo scope**：2D 或轻量 3D；8–12 周可玩 demo；**排除** 3A/强联机/live ops/大量手工关卡
- 手游/主机：仅 **跨平台参照** 一句
- 联网补商店页：与 A4 **合计 ≤3 次/周**

### A1 准入清单（中等）

1. 原文摩擦句 ≤40 字 + URL + 日期
2. 可识别人群 + 场景
3. Vibe Coding 适配说明
4. 单源可进 A1，须标「**单源，待交叉验证**」
5. 每周 A1 **2–3 簇**或写「本周无强证据簇」
